743. 网络延迟时间

有 n 个网络节点，标记为 1 到 n。

给你一个列表 times，表示信号经过 有向 边的传递时间。 times[i] = (ui, vi, wi)，其中 ui 是源节点，vi 是目标节点， wi 是一个信号从源节点传递到目标节点的时间。

现在，从某个节点 K 发出一个信号。需要多久才能使所有节点都收到信号？如果不能使所有节点收到信号，返回 -1 。

朴素Bellman Ford

执行 N - 1N−1 次松弛操作即可保证所有边达到最小值

松弛操作原理是 三角形 两边之和大于第三边，举个🌰：求源点 A 到其他结点的最短距离，有两个结点 B 和 C 与源点 A 的距离为 x, y, 若 B 到 C 之间有一条边 z ，那么此时可以考虑通过 B 到达 C, 距离为 x + z，若 x + z < y，说明通过 B 到达 C 距离更短，就可以更新 C 与 源点 A 的最短距离

拓展：因为松弛操作只会发生在上一轮松弛过的结点的边上，所以可以维护一个队列保存松弛过的结点，该方法即 SPFA算法，可以避免遍历所有边执行松弛操作，但最坏情况下还是会退化成朴素BF
